# Quick Start

This guide will help you get familiar with Shadcn Admin's structure, customize the dashboard, and start building your own admin interface.

<Note>
  Make sure you've completed the [installation](/installation) before proceeding.
</Note>

## Project Overview

After starting the development server with `pnpm run dev`, open your browser to `http://localhost:5173`. You'll see a fully functional admin dashboard with:

- A responsive sidebar with navigation
- Dashboard page with analytics charts
- User and task management tables
- Settings pages
- Authentication pages
- Error page templates

## Understanding the Structure

Shadcn Admin follows a feature-based architecture:

```plaintext
src/
├── components/
│   ├── ui/                    # Shadcn UI components
│   ├── app-sidebar.tsx        # Main navigation sidebar
│   ├── breadcrumb-provider.tsx
│   ├── data-table/            # Reusable table components
│   └── ...
├── features/
│   ├── dashboard/             # Dashboard analytics
│   ├── users/                 # User management
│   ├── tasks/                 # Task management
│   ├── settings/              # Settings pages
│   ├── auth/                  # Authentication
│   └── errors/                # Error pages
├── routes/                    # TanStack Router routes
├── hooks/                     # Custom React hooks
├── lib/                       # Utilities and helpers
├── stores/                    # Zustand state stores
├── context/                   # React context providers
└── main.tsx                   # Application entry point
```

## Customizing the Dashboard

### 1. Update Sidebar Navigation

Edit `src/components/app-sidebar.tsx` to customize navigation items:

```typescript src/components/app-sidebar.tsx
const navItems = [
  {
    title: 'Dashboard',
    url: '/',
    icon: LayoutGrid,
  },
  {
    title: 'Users',
    url: '/users',
    icon: Users,
  },
  // Add your own navigation items
  {
    title: 'Products',
    url: '/products',
    icon: Package,
  },
]
```

```css src/styles/theme.css

    --foreground: 222.2 84% 4.9%;
    --primary: 222.2 47.4% 11.2%;
    --primary-foreground: 210 40% 98%;
    /* ... more theme variables */
  }
  
  .dark {
    --background: 222.2 84% 4.9%;
    --foreground: 210 40% 98%;
    /* ... dark theme variables */
  }

```

### 3. Add New UI Components

Shadcn Admin uses Shadcn UI. Add new components using the CLI:

```bash
npx shadcn@latest add [component-name]
```

Example:

```bash
npx shadcn@latest add badge
```

<Warning>
  Some components are customized for RTL support. Check the [component customizations list](/introduction#component-customizations) before updating.
</Warning>

## Working with Forms

Shadcn Admin uses React Hook Form with Zod validation. Here's a quick example:

```typescript
import { useForm } from 'react-hook-form'
import { zodResolver } from '@hookform/resolvers/zod'
import * as z from 'zod'
import { Form, FormField, FormItem, FormLabel, FormControl } from '@/components/ui/form'
import { Input } from '@/components/ui/input'
import { Button } from '@/components/ui/button'

const formSchema = z.object({
  name: z.string().min(2, 'Name must be at least 2 characters'),
  email: z.string().email('Invalid email address'),
})

export function MyForm() {
  const form = useForm({
    resolver: zodResolver(formSchema),
    defaultValues: {
      name: '',
      email: '',
    },
  })

  function onSubmit(values: z.infer<typeof formSchema>) {
    console.log(values)
  }

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit(onSubmit)} className="space-y-4">
        <FormField
          control={form.control}
          name="name"
          render={({ field }) => (
            <FormItem>
              <FormLabel>Name</FormLabel>
              <FormControl>
                <Input {...field} />
              </FormControl>
            </FormItem>
          )}
        />
        <Button type="submit">Submit</Button>
      </form>
    </Form>
  )
}
```

## State Management

The project uses Zustand for global state. Example auth store at `src/stores/auth-store.ts`:

```typescript
import { create } from 'zustand'

interface AuthStore {
  user: User | null
  setUser: (user: User) => void
  logout: () => void
}

export const useAuthStore = create<AuthStore>((set) => ({
  user: null,
  setUser: (user) => set({ user }),
  logout: () => set({ user: null }),
}))
```

Use it in your components:

```typescript
import { useAuthStore } from '@/stores/auth-store'

function MyComponent() {
  const user = useAuthStore((state) => state.user)
  const logout = useAuthStore((state) => state.logout)
  
  // ...
}
```

## Data Fetching with TanStack Query

The app is configured with TanStack Query in `src/main.tsx`:

```typescript src/main.tsx
const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      refetchOnWindowFocus: import.meta.env.PROD,
      staleTime: 10 * 1000, // 10s
    },
  },
})
```

Example query usage:

```typescript
import { useQuery } from '@tanstack/react-query'
import axios from 'axios'

function useUsers() {
  return useQuery({
    queryKey: ['users'],
    queryFn: async () => {
      const { data } = await axios.get('/api/users')
      return data
    },
  })
}
```

## Tips for Success

<AccordionGroup>
  <Accordion title="Use TypeScript">
    Take advantage of full type safety. The project includes types for TanStack Router, ensuring type-safe navigation and route parameters.
  </Accordion>

  <Accordion title="Leverage Path Aliases">
    Use `@/` imports instead of relative paths:
    ```typescript
    // Good
    import { Button } from '@/components/ui/button'
    
    // Avoid
    import { Button } from '../../../components/ui/button'
    ```
  </Accordion>

  <Accordion title="Follow the Feature Structure">
    Keep related components, hooks, and utilities within their feature directories for better organization and maintainability.
  </Accordion>

  <Accordion title="Check Existing Components First">
    Before creating new components, browse `src/components/ui/` and `src/components/` to see what's already available.
  </Accordion>
</AccordionGroup>
