Shadcn Admin provides flexible layout options including sidebar variants, collapsible modes, and responsive behavior.

## Layout Provider

The `LayoutProvider` manages layout state including sidebar variant and collapsible mode.

### Setup

The LayoutProvider is configured in the authenticated layout:

```tsx src/components/layout/authenticated-layout.tsx
import { LayoutProvider } from '@/context/layout-provider'
import { SidebarProvider, SidebarInset } from '@/components/ui/sidebar'
import { AppSidebar } from '@/components/layout/app-sidebar'

export function AuthenticatedLayout({ children }) {
  const defaultOpen = getCookie('sidebar_state') !== 'false'
  
  return (
    <LayoutProvider>
      <SidebarProvider defaultOpen={defaultOpen}>
        <AppSidebar />
        <SidebarInset>
          {children}
        </SidebarInset>
      </SidebarProvider>
    </LayoutProvider>
  )
}
```

## Using the Layout Hook

Access and control layout settings:

```tsx
import { useLayout } from '@/context/layout-provider'

function LayoutControls() {
  const { variant, setVariant, collapsible, setCollapsible } = useLayout()
  
  return (
    <div>
      <button onClick={() => setVariant('floating')}>
        Floating Sidebar
      </button>
      <button onClick={() => setCollapsible('icon')}>
        Icon Mode
      </button>
    </div>
  )
}
```

## Layout API

The `useLayout` hook provides:

| Property | Type | Description |
|----------|------|-------------|
| `variant` | `'inset' \| 'sidebar' \| 'floating'` | Sidebar variant style |
| `setVariant` | `(variant: Variant) => void` | Update sidebar variant |
| `defaultVariant` | `Variant` | Default variant (`'inset'`) |
| `collapsible` | `'offcanvas' \| 'icon' \| 'none'` | Collapsible behavior |
| `setCollapsible` | `(collapsible: Collapsible) => void` | Update collapsible mode |
| `defaultCollapsible` | `Collapsible` | Default mode (`'icon'`) |
| `resetLayout` | `() => void` | Reset all layout settings |

## Sidebar Variants

Shadcn Admin supports three sidebar variants:

<Tabs>
  <Tab title="Inset">
    The inset variant creates a bordered sidebar with padding from the edges.
    
    ```tsx
    const { setVariant } = useLayout()
    setVariant('inset')
    ```
    
    **Characteristics:**
    - Sidebar has visual separation from content
    - Padding around the sidebar
    - Background color differs from main content
    - Default variant
  </Tab>
  
  <Tab title="Floating">
    The floating variant creates an elevated sidebar that appears to float over the content.
    
    ```tsx
    const { setVariant } = useLayout()
    setVariant('floating')
    ```
    
    **Characteristics:**
    - Sidebar appears to float with shadow
    - No padding from edges
    - Overlays content on mobile
  </Tab>
  
  <Tab title="Sidebar">
    The standard sidebar variant with no special styling.
    
    ```tsx
    const { setVariant } = useLayout()
    setVariant('sidebar')
    ```
    
    **Characteristics:**
    - Edge-to-edge sidebar
    - No elevation or padding
    - Traditional sidebar appearance
  </Tab>
</Tabs>

### Applying Variants

Variants are applied to the Sidebar component:

```tsx src/components/layout/app-sidebar.tsx
import { useLayout } from '@/context/layout-provider'
import { Sidebar } from '@/components/ui/sidebar'

export function AppSidebar() {
  const { collapsible, variant } = useLayout()
  
  return (
    <Sidebar collapsible={collapsible} variant={variant}>
      <SidebarHeader>
        <TeamSwitcher teams={sidebarData.teams} />
      </SidebarHeader>
      <SidebarContent>
        {/* Navigation */}
      </SidebarContent>
      <SidebarFooter>
        <NavUser user={sidebarData.user} />
      </SidebarFooter>
      <SidebarRail />
    </Sidebar>
  )
}
```

## Collapsible Modes

<Steps>

<Step title="Default (None)">
Sidebar remains fully expanded and cannot be collapsed.

```tsx
const { setCollapsible } = useLayout()
setCollapsible('none')
```

**Use case:** When you want a persistent sidebar that's always visible.
</Step>

<Step title="Icon Mode">
Sidebar collapses to show only icons, hiding text labels.

```tsx
const { setCollapsible } = useLayout()
setCollapsible('icon')
```

**Features:**
- Icons remain visible
- Text labels hidden
- Tooltips show on hover
- Default collapsible mode
</Step>

<Step title="Offcanvas">
Sidebar completely hides from view when collapsed.

```tsx
const { setCollapsible } = useLayout()
setCollapsible('offcanvas')
```

**Features:**
- Entire sidebar hidden
- Maximum content space
- Mobile-friendly
</Step>

</Steps>

## Sidebar State Management

The sidebar open/closed state is managed separately from the layout provider:

```tsx
import { useSidebar } from '@/components/ui/sidebar'

function SidebarToggle() {
  const { open, setOpen, toggleSidebar } = useSidebar()
  
  return (
    <button onClick={toggleSidebar}>
      {open ? 'Close' : 'Open'} Sidebar
    </button>
  )
}
```

### Sidebar Cookie Storage

The sidebar state is persisted in cookies:

```tsx src/components/ui/sidebar.tsx
const SIDEBAR_COOKIE_NAME = 'sidebar_state'
const SIDEBAR_COOKIE_MAX_AGE = 60 * 60 * 24 * 7 // 7 days

const setOpen = (value: boolean) => {
  _setOpen(value)
  document.cookie = `${SIDEBAR_COOKIE_NAME}=${value}; path=/; max-age=${SIDEBAR_COOKIE_MAX_AGE}`
}
```

## Layout Configuration

Layout settings are persisted in cookies:

```tsx src/context/layout-provider.tsx
const LAYOUT_COLLAPSIBLE_COOKIE_NAME = 'layout_collapsible'
const LAYOUT_VARIANT_COOKIE_NAME = 'layout_variant'
const LAYOUT_COOKIE_MAX_AGE = 60 * 60 * 24 * 7 // 7 days

const setVariant = (newVariant: Variant) => {
  _setVariant(newVariant)
  setCookie(LAYOUT_VARIANT_COOKIE_NAME, newVariant, LAYOUT_COOKIE_MAX_AGE)
}

const setCollapsible = (newCollapsible: Collapsible) => {
  _setCollapsible(newCollapsible)
  setCookie(LAYOUT_COLLAPSIBLE_COOKIE_NAME, newCollapsible, LAYOUT_COOKIE_MAX_AGE)
}
```

## Responsive Layout

### Mobile Behavior

The sidebar automatically adapts to mobile:

```tsx src/components/ui/sidebar.tsx
const SIDEBAR_WIDTH = '16rem'        // Desktop
const SIDEBAR_WIDTH_MOBILE = '18rem' // Mobile
const SIDEBAR_WIDTH_ICON = '3rem'    // Icon mode

function SidebarProvider({ children }) {
  const isMobile = useIsMobile()
  const [openMobile, setOpenMobile] = useState(false)
  
  // Mobile uses sheet overlay instead of inline sidebar
  // ...
}
```

### Container Queries

The content area uses container queries for responsive design:

```tsx src/components/layout/authenticated-layout.tsx
<SidebarInset
  className={
    '@container/content ' +
    'has-data-[layout=fixed]:h-svh ' +
    'peer-data-[variant=inset]:has-data-[layout=fixed]:h-[calc(100svh-(var(--spacing)*4))]'
  }
>
  {children}
</SidebarInset>
```

Content can use container query variants:

```tsx
<div className="grid grid-cols-1 @md:grid-cols-2 @lg:grid-cols-3">
  {/* Responsive grid based on container width */}
</div>
```

## Keyboard Shortcuts

Toggle sidebar with keyboard:

```tsx src/components/ui/sidebar.tsx
const SIDEBAR_KEYBOARD_SHORTCUT = 'b'

useEffect(() => {
  const handleKeyDown = (event: KeyboardEvent) => {
    if (
      event.key === SIDEBAR_KEYBOARD_SHORTCUT &&
      (event.metaKey || event.ctrlKey)
    ) {
      event.preventDefault()
      toggleSidebar()
    }
  }
  
  window.addEventListener('keydown', handleKeyDown)
  return () => window.removeEventListener('keydown', handleKeyDown)
}, [toggleSidebar])
```

**Shortcut:** <kbd>Cmd/Ctrl</kbd> + <kbd>B</kbd>

## Configuration UI

The ConfigDrawer provides a visual interface for layout settings:

```tsx src/components/config-drawer.tsx
function LayoutConfig() {
  const { open, setOpen } = useSidebar()
  const { defaultCollapsible, collapsible, setCollapsible } = useLayout()

  const radioState = open ? 'default' : collapsible

  return (
    <Radio
      value={radioState}
      onValueChange={(v) => {
        if (v === 'default') {
          setOpen(true)
          return
        }
        setOpen(false)
        setCollapsible(v as Collapsible)
      }}
    >
      <Item value="default" label="Default">
        <IconLayoutDefault />
      </Item>
      <Item value="icon" label="Compact">
        <IconLayoutCompact />
      </Item>
      <Item value="offcanvas" label="Full layout">
        <IconLayoutFull />
      </Item>
    </Radio>
  )
}
```

## Advanced Layout Patterns

### Fixed Height Layout

For layouts with fixed height content:

```tsx
<div data-layout="fixed" className="h-full overflow-hidden">
  <Header />
  <Main className="overflow-auto" />
  <Footer />
</div>
```

The layout automatically adjusts:

```tsx
<SidebarInset
  className="has-data-[layout=fixed]:h-svh"
>
  {/* Content */}
</SidebarInset>
```

### Custom Variants

Create custom sidebar variants by extending the types:

```tsx
// Add to layout-provider.tsx
export type Variant = 'inset' | 'sidebar' | 'floating' | 'custom'

// Apply custom styling
<Sidebar 
  variant="custom"
  className="border-l-4 border-primary"
/>
```

## Best Practices

<Steps>

<Step title="Respect User Preferences">
Always persist layout choices so users don't need to reconfigure on every visit:

```tsx
const { variant } = useLayout()
// variant is automatically loaded from cookies
```
</Step>

<Step title="Mobile-First Approach">
Test your layout on mobile devices. The sidebar automatically becomes an overlay on mobile:

```tsx
const isMobile = useIsMobile()
// Sidebar automatically uses Sheet on mobile
```
</Step>

<Step title="Provide Visual Feedback">
Show users their current layout state in the configuration UI:

```tsx
const { variant, collapsible } = useLayout()
// Display current settings in ConfigDrawer
```
</Step>

<Step title="Use Container Queries">
Make content responsive to sidebar state, not just viewport:

```tsx
<div className="@container/content">
  <div className="grid @md:grid-cols-2">
    {/* Adapts to container width */}
  </div>
</div>
```
</Step>

</Steps>

<Note>
Layout settings are stored in cookies with a 7-day expiration, while theme settings persist for 1 year. This ensures layout preferences are remembered across sessions.
</Note>

<Warning>
When using `collapsible="none"`, ensure your layout has enough space for the full sidebar on smaller screens, or provide alternative navigation for mobile users.
</Warning>
