## Page Structure

All page templates in Shadcn Admin follow a consistent architectural pattern:

### Route Definition

Pages are defined using TanStack Router's file-based routing system:

```tsx
// src/routes/_authenticated/index.tsx
import { createFileRoute } from '@tanstack/react-router'
import { Dashboard } from '@/features/dashboard'

export const Route = createFileRoute('/_authenticated/')({\n  component: Dashboard,
})
```

### Feature-Based Organization

Page components are organized in the `src/features/` directory, separating routing logic from UI implementation:

```
src/
├── routes/              # Route definitions
│   ├── _authenticated/  # Protected routes
│   ├── (auth)/         # Auth routes
│   └── (errors)/       # Error pages
└── features/           # Page implementations
    ├── dashboard/
    ├── auth/
    ├── settings/
    └── errors/
```

## Authentication

Pages are organized into two main groups:

<Tabs>
  <Tab title="Protected Routes">
    Routes under `_authenticated/` require authentication:
    - Dashboard (`/`)
    - Tasks (`/tasks`)
    - Users (`/users`)
    - Settings (`/settings/*`)
    - Chats (`/chats`)
    - Apps (`/apps`)
  </Tab>
  <Tab title="Public Routes">
    Routes under `(auth)/` and `(errors)/` are publicly accessible:
    - Sign In (`/sign-in`)
    - Sign Up (`/sign-up`)
    - Forgot Password (`/forgot-password`)
    - OTP Verification (`/otp`)
    - Error pages (`/401`, `/403`, `/404`, `/500`, `/503`)
  </Tab>
</Tabs>

## Search Params & Filtering

Many pages support URL-based filtering and pagination using Zod schemas:

```tsx
// src/routes/_authenticated/tasks/index.tsx
const taskSearchSchema = z.object({
  page: z.number().optional().catch(1),
  pageSize: z.number().optional().catch(10),
  status: z.array(z.enum(['todo', 'in-progress', 'done'])).optional(),
  priority: z.array(z.enum(['low', 'medium', 'high'])).optional(),
  filter: z.string().optional().catch(''),
})

export const Route = createFileRoute('/_authenticated/tasks/')({\n  validateSearch: taskSearchSchema,
  component: Tasks,
})
```

<Note>
URL search params are type-safe and validated at runtime using Zod schemas.
</Note>
