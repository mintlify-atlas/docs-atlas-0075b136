Shadcn Admin includes a comprehensive settings section with multiple pages for managing user preferences, account details, and application appearance.

## Overview

The settings section uses a shared layout with sidebar navigation and consistent form patterns across all pages.

**Location:** `src/features/settings/`  
**Routes:** `/_authenticated/settings/` directory

## Profile Settings

Manage personal information and public profile.

**Route:** `/settings` (default settings page)  
**File:** `src/features/settings/profile/index.tsx`

### Implementation

```tsx
// src/routes/_authenticated/settings/index.tsx
import { createFileRoute } from '@tanstack/react-router'
import { SettingsProfile } from '@/features/settings/profile'

export const Route = createFileRoute('/_authenticated/settings/')({\n  component: SettingsProfile,
})
```

### Component Structure

```tsx
// src/features/settings/profile/index.tsx
import { ContentSection } from '../components/content-section'
import { ProfileForm } from './profile-form'

export function SettingsProfile() {
  return (
    <ContentSection
      title='Profile'
      desc='This is how others will see you on the site.'
    >
      <ProfileForm />
    </ContentSection>
  )
}
```

### Profile Form Fields

- **Username:** Display name
- **Email:** Contact email (read-only if verified)
- **Bio:** Short description
- **URLs:** Social links and website
- **Avatar:** Profile picture upload

## Form Patterns

All settings forms follow consistent patterns:

### Form Structure

```tsx
import { zodResolver } from '@hookform/resolvers/zod'
import { useForm } from 'react-hook-form'
import * as z from 'zod'
import {
  Form,
  FormControl,
  FormDescription,
  FormField,
  FormItem,
  FormLabel,
  FormMessage,
} from '@/components/ui/form'
import { Button } from '@/components/ui/button'
import { Input } from '@/components/ui/input'

const formSchema = z.object({
  // Define your schema
})

export function YourForm() {
  const form = useForm<z.infer<typeof formSchema>>({
    resolver: zodResolver(formSchema),
    defaultValues: {
      // Default values
    },
  })

  function onSubmit(values: z.infer<typeof formSchema>) {
    // Handle form submission
  }

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit(onSubmit)} className='space-y-8'>
        <FormField
          control={form.control}
          name='fieldName'
          render={({ field }) => (
            <FormItem>
              <FormLabel>Field Label</FormLabel>
              <FormControl>
                <Input {...field} />
              </FormControl>
              <FormDescription>Field description</FormDescription>
              <FormMessage />
            </FormItem>
          )}
        />
        <Button type='submit'>Save changes</Button>
      </form>
    </Form>
  )
}
```

### Validation

All forms use Zod for schema validation:

```tsx
const profileFormSchema = z.object({
  username: z
    .string()
    .min(2, 'Username must be at least 2 characters.')
    .max(30, 'Username must not be longer than 30 characters.'),
  email: z.string().email('Invalid email address.'),
  bio: z.string().max(160).optional(),
})
```

Settings pages are fully responsive:

- **Mobile:** Stacked layout, collapsible sidebar
- **Tablet:** Side-by-side with compact sidebar
- **Desktop:** Full sidebar with expanded navigation

<Info>
On mobile devices, the settings navigation appears as a dropdown menu instead of a sidebar.
</Info>

## Adding Custom Settings Pages

To add a new settings page:

### 1. Create the Route

```tsx
// src/routes/_authenticated/settings/custom.tsx
import { createFileRoute } from '@tanstack/react-router'
import { SettingsCustom } from '@/features/settings/custom'

export const Route = createFileRoute('/_authenticated/settings/custom')({\n  component: SettingsCustom,
})
```

### 2. Create the Feature Component

```tsx
// src/features/settings/custom/index.tsx
import { ContentSection } from '../components/content-section'
import { CustomForm } from './custom-form'

export function SettingsCustom() {
  return (
    <ContentSection
      title='Custom Settings'
      desc='Your custom settings description.'
    >
      <CustomForm />
    </ContentSection>
  )
}
```

### 3. Create the Form

```tsx
// src/features/settings/custom/custom-form.tsx
import { zodResolver } from '@hookform/resolvers/zod'
import { useForm } from 'react-hook-form'
import * as z from 'zod'
// ... implement your form
```

### 4. Update Navigation

Add your new page to the settings navigation configuration.
