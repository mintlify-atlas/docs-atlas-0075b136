## Architecture

### Component Location

Components are organized in the following structure:

```
src/components/
├── ui/                    # Base UI components (30+ components)
├── data-table/            # Data table components
├── layout/                # Layout and navigation components
└── [feature-components]   # Feature-specific components
```

### Design Principles

All components follow these principles:

- **Type-Safe**: Written in TypeScript with full type definitions
- **Composable**: Built with composition patterns for maximum flexibility
- **Accessible**: ARIA-compliant with keyboard navigation support
- **Customizable**: Styling via Tailwind CSS with theme support
- **Data Attributes**: Using `data-slot` attributes for consistent styling

## Specialized Components

### Data Display

- **Table** - Basic table component with styling
- **Calendar** - Date picker calendar
- **Avatar** - User profile images
- **Scroll Area** - Custom scrollable containers

### Custom Components

Beyond the base UI components, Shadcn Admin includes:

- **DatePicker** - Calendar-based date selection
- **PasswordInput** - Password field with show/hide toggle
- **LongText** - Text truncation with expand functionality
- **ConfigDrawer** - Settings and configuration panel
- **ProfileDropdown** - User profile menu
- **ThemeSwitch** - Light/dark mode toggle
- **Search** - Global search component

## Data Table System

The data table components provide:

- **Pagination** - Page navigation with customizable rows per page
- **Column Header** - Sortable column headers
- **Faceted Filter** - Multi-select filtering
- **View Options** - Column visibility toggle
- **Toolbar** - Search and filter controls
- **Bulk Actions** - Actions on selected rows

See [Data Table](/components/data-table) for detailed documentation.

## Layout Components

Layout components handle navigation and page structure:

- **AppSidebar** - Main application sidebar
- **NavGroup** - Grouped navigation items
- **TeamSwitcher** - Multi-team dropdown selector
- **NavUser** - User profile in sidebar
- **Header** - Page header component
- **Main** - Main content wrapper

See [Sidebar](/components/sidebar) for detailed documentation.

## Form Integration

Form components integrate React Hook Form with Zod validation:

- **Form** - Form provider wrapper
- **FormField** - Field wrapper with validation
- **FormItem** - Field layout container
- **FormLabel** - Accessible form labels
- **FormControl** - Input control wrapper
- **FormMessage** - Validation error messages

See [Forms](/components/forms) for detailed documentation.

## Usage Pattern

All components follow a consistent import and usage pattern:

```tsx
import { Button } from '@/components/ui/button'
import { Card, CardHeader, CardTitle, CardContent } from '@/components/ui/card'

export function Example() {
  return (
    <Card>
      <CardHeader>
        <CardTitle>Example Card</CardTitle>
      </CardHeader>
      <CardContent>
        <Button>Click Me</Button>
      </CardContent>
    </Card>
  )
}
```
