Shadcn Admin includes 30+ pre-built UI components located in `src/components/ui/`. All components are built with Radix UI primitives, styled with Tailwind CSS, and fully type-safe.

## Button Component

The Button component supports multiple variants and sizes with icon integration.

### Props

<ParamField path="variant" type="string" default="default">
  Button style variant: `default`, `destructive`, `outline`, `secondary`, `ghost`, or `link`
</ParamField>

<ParamField path="size" type="string" default="default">
  Button size: `default`, `sm`, `lg`, or `icon`
</ParamField>

<ParamField path="asChild" type="boolean" default="false">
  Render as a child component (useful for links)
</ParamField>

### Basic Usage

```tsx
import { Button } from '@/components/ui/button'

export function Example() {
  return (
    <div className="flex gap-2">
      <Button variant="default">Default</Button>
      <Button variant="destructive">Delete</Button>
      <Button variant="outline">Cancel</Button>
      <Button variant="ghost">Ghost</Button>
      <Button variant="link">Link</Button>
    </div>
  )
}
```

### With Icons

```tsx
import { Plus, Trash2 } from 'lucide-react'
import { Button } from '@/components/ui/button'

export function ButtonWithIcon() {
  return (
    <div className="flex gap-2">
      <Button>
        <Plus />
        Add User
      </Button>
      <Button variant="destructive" size="icon">
        <Trash2 />
      </Button>
    </div>
  )
}
```

### Variants Reference

From `src/components/ui/button.tsx:6-35`:

- **default**: Primary button with solid background
- **destructive**: Danger/delete actions with red styling
- **outline**: Bordered button with transparent background
- **secondary**: Secondary actions with muted styling
- **ghost**: Minimal button with hover effect only
- **link**: Text-only button with underline on hover

## Card Component

Card is a composable container component with multiple sub-components.

### Sub-Components

- **Card** - Main container
- **CardHeader** - Header section with title and description
- **CardTitle** - Card title
- **CardDescription** - Subtitle or description
- **CardAction** - Action button area in header
- **CardContent** - Main content area
- **CardFooter** - Footer section

### Basic Usage

```tsx
import {
  Card,
  CardHeader,
  CardTitle,
  CardDescription,
  CardContent,
  CardFooter,
} from '@/components/ui/card'
import { Button } from '@/components/ui/button'

export function Example() {
  return (
    <Card>
      <CardHeader>
        <CardTitle>User Statistics</CardTitle>
        <CardDescription>
          Overview of user activity for the last 30 days
        </CardDescription>
      </CardHeader>
      <CardContent>
        <p>1,234 active users</p>
      </CardContent>
      <CardFooter>
        <Button>View Details</Button>
      </CardFooter>
    </Card>
  )
}
```

### With Action Button

### DatePicker

Calendar-based date picker from `src/components/date-picker.tsx`.

<ParamField path="selected" type="Date | undefined" required>
  Currently selected date
</ParamField>

<ParamField path="onSelect" type="(date: Date | undefined) => void" required>
  Callback when date is selected
</ParamField>

<ParamField path="placeholder" type="string" default="Pick a date">
  Placeholder text when no date is selected
</ParamField>

```tsx
import { useState } from 'react'
import { DatePicker } from '@/components/date-picker'

export function Example() {
  const [date, setDate] = useState<Date | undefined>()

  return (
    <DatePicker
      selected={date}
      onSelect={setDate}
      placeholder="Select a date"
    />
  )
}
```
